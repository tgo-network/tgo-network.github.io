# TGO Network 内容工作流

## 1. 目的

本文档定义后台内容如何从创建走向公开发布。

当前主要适用于：

- 首页区块
- 分会与董事会信息
- 成员资料
- 文章
- 活动
- 单页内容（`/join`、`/about`）

## 2. 工作流原则

- 公开内容必须有明确发布状态
- 草稿不得出现在公开 API 中
- 当前阶段优先简单可用，不引入复杂审批引擎
- 发布规则应由 API 统一控制，而不是由前端自行决定

## 3. 当前推荐状态模型

对大多数公开实体，优先使用：

- `draft`
- `published`
- `archived`

对文章与活动，可额外支持：

- `scheduled`

含义：

- `draft`：编辑中，不公开
- `scheduled`：已准备好，在未来时间自动公开
- `published`：公开可见
- `archived`：停止公开，但保留后台记录

## 4. 当前阶段的最小发布流程

推荐统一为：

1. 创建草稿
2. 编辑内容
3. 完成必要校验
4. 发布或定时发布
5. 不再使用时归档

## 5. 各内容类型规则

### 首页区块

首页不是单篇文章，而是结构化区块组合。

当前建议区块：

- Hero
- 组织介绍
- 覆盖人群
- 数据指标
- 精彩活动 / 精选文章
- 加入 CTA

管理规则：

- 每个区块可单独启停
- 首页区块更新后应有明确的发布时间点
- 首页引用的文章、活动、分会只允许选择已发布内容

### 分会与董事会

分会发布前至少应具备：

- 分会名称
- 城市或区域
- 分会简介

董事会展示建议至少具备：

- 姓名
- 公司
- 职称
- 头像（推荐）

当前建议：

- 分会与董事会信息由成员域维护
- 分会页面只展示 `published` 状态的分会
- 分会下的董事会成员也必须是可公开状态

### 成员

成员公开前至少应具备：

- 姓名
- 公司
- 职称
- 加入时间

推荐补充：

- 头像
- 个人简介
- 所属分会

公开规则：

- 成员列表只返回可公开成员
- 成员详情读取更完整资料
- 隐藏成员不得出现在列表中

### 文章

文章发布前至少应具备：

- 标题
- `slug`
- 摘要
- 正文
- 作者名

推荐补充：

- 封面图
- SEO 标题与描述

文章规则：

- `scheduled` 文章只有到时后才进入公开接口
- 发布后如修改 `slug`，应记录审计日志
- 归档文章应立即从公开列表中移除

### 活动

活动发布前至少应具备：

- 标题
- `slug`
- 分会或城市
- 开始时间
- 结束时间
- 地点
- 简介或正文

活动有两套状态：

- 内容状态：`draft` / `published` / `archived`
- 报名状态：`closed` / `open` / `waitlist` / `external`

重要区分：

- 内容发布决定页面是否公开可见
- 报名状态决定是否允许提交报名

当前报名流转：

1. 前台在活动详情页提交表单
2. API 写入 `event_registrations`
3. 后台活动报名审核页查看记录
4. 工作人员更新状态并填写审核备注

补充约束：

- 当前阶段活动报名先开放提交
- 是否通过、是否符合活动要求，由工作人员审核判断

### 单页内容

当前单页包括：

- `/join`
- `/about`

`join` 页发布前建议具备：

- 申请条件
- 申请流程
- FAQ
- 跳转申请按钮

`about` 页发布前建议具备：

- 组织背景
- 活动形式
- 加入方式说明

## 6. 预览策略

当前阶段建议：

- 预览只对已登录工作人员开放
- 预览内容通过后台权限 API 获取
- 不允许通过放宽公开接口过滤来实现预览

## 7. SEO 规则

每个公开页面类型都建议支持：

- `seo_title`
- `seo_description`
- canonical URL
- Open Graph 图片

如果未配置 SEO 字段，可退回到标题与摘要生成。

## 8. 负责人归属建议

- 内容编辑负责：首页、加入页、关于页、文章
- 活动运营负责：活动与报名审核
- 成员运营负责：分会、董事会、成员资料
- 审核人员负责：加入申请处理
- 超级管理员负责：跨域覆盖与最终兜底

## 9. 定时发布

当前阶段不必做复杂编排。

建议仅在以下类型支持：

- 文章
- 活动（如确有需要）

通过内部任务定时把到点记录从 `scheduled` 切换到 `published`。

## 10. 当前明确不做的工作流能力

- 多级审批流
- 协同编辑
- 可视化差异比较
- 多语言本地化工作流
- 成员自助编辑资料审核流
