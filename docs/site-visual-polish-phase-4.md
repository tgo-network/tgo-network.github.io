# 前台统一视觉精修（M3-4）

## 目标

在 `M3-3` 已经完成首页节奏优化后，本轮继续按照最新产品反馈收敛首页，重点解决四个问题：

- Hero 文案偏多，右侧结构过重
- 首页仍然出现了分会展示，不符合当前首页范围
- `TGO 介绍` 区文案与布局偏散，尤其“精彩活动”部分结构杂乱
- 成员区需要从“名单展示”切换为“成员推荐语”，与文章区分开承担不同职责

## 本轮完成

### 1. Hero 改成更直接的品牌入口

文件：`apps/site/src/pages/index.astro`

- 将首页主标题收敛为更短的品牌表达：聚焦“科技领导者”“高质量学习社区”
- Hero 摘要改成单段说明，只保留社区定位，不再重复解释页面阅读顺序
- Hero 右侧从多层说明卡改成两组轻量信息：`适合人群` 与 `同侪学习 / 公开活动`
- 进一步移除 Hero 中的成员 / 活动 / 文章数字卡片，避免次要运营信息占据首页最强位置

### 2. 首页移除分会展示

文件：`apps/site/src/pages/index.astro`

- 删除首页中的 `分会董事会` 展示段落
- 首页不再承担组织结构目录页职责，分会信息保留在独立的 `/branches` 页面中承接
- 首页信息路径收敛为 `品牌 -> 组织说明 -> 成员推荐 -> 活动 -> 文章 -> 加入申请`

### 3. Intro 重写为更简洁的组织说明

文件：`apps/site/src/pages/index.astro`

本轮 intro 文案参考了 `https://tgo.infoq.cn/` 公开可见的品牌描述与站点文案方向，并做了压缩整理。

- 以“技术、资源与知识连接起来，产生更大价值”这类目标站核心表达为基础，重写首页组织说明
- 将原先复杂的 `组织定位 / 覆盖人群 / 内容定位 / 精彩活动` 混合结构，改成：
  - 一个主说明面板
  - 三张简洁说明卡：`社区定位 / 小组活动 / 公开学习`
- 删除原先层层嵌套的“精彩活动小卡片”，让 intro 更像组织介绍，而不是活动类型清单

### 4. 成员与文章彻底分开

文件：`apps/site/src/pages/index.astro`

- 成员区改为 `成员推荐`，通过成员头像、身份与推荐语建立首页信任感
- 成员区不再承担成员列表预览职责，而是突出“为什么值得加入”
- 文章区恢复为独立段落，用于承接活动之后的观点与经验沉淀
- 成员和文章不再被压缩进同一个证明模块，各自承担清晰职责
- 首页中的辅助性 `section-note` 说明文案进一步移除，避免把开发阶段的解释信息暴露给前台访客

## 当前结果

完成本轮后，首页已经从“内容完整但解释过多”进一步推进到“更接近正式品牌首页”的状态。

当前最明显的收敛点包括：

- Hero 更短、更直接，不再有明显的说明页口吻
- 首页已经移除分会展示，结构重心回到品牌、信任、活动与转化
- Intro 不再出现杂乱的活动说明拼盘，而是回到简洁的组织介绍
- 成员区从名单展示切换为推荐语展示，首页信任感明显增强

## 补充收口

在后续回看截图时，又针对首页第一屏和 `TGO 介绍` 区做了一轮轻量收口，文件仍然是 `apps/site/src/pages/index.astro`：

- Hero 主标题改成双行排版，避免中文标题在桌面和移动端都被压成过窄的竖排感
- 去掉 Hero 中重复的辅助短标题，只保留组织名、主标题、摘要和 CTA
- Hero 右侧把长标题式的人群说明改成更短的标题加角色标签，信息更容易扫读
- `TGO 介绍` 主标题与主说明再次压缩，减少首页上半部分的字量密度
- 对 Intro 主卡和说明卡做了轻微视觉强化，让首页前两屏更稳定、正式

## 下一轮建议

如果继续做首页最后一轮收尾，建议优先考虑：

1. 将成员推荐语进一步替换为真实成员口碑内容，减少演示数据感
2. 根据正式运营内容决定文章区是保留 3 条还是继续压缩为 2 条
3. 开始进入全站正式文案统一与素材替换，而不是继续大幅调整首页结构
