# 前台统一视觉精修（M3-3）

## 目标

在 `M3-2` 已经完成列表页、说明页和申请页的视觉收敛后，本轮继续处理两类剩余问题：

- 首页虽然内容结构已经完整，但长页面中相邻内容区块的节奏切换还不够明确
- 成员 / 活动 / 文章详情页的信息已经齐全，但正文、侧栏和补充说明区之间的阅读重心还可以更清楚

## 本轮完成

### 1. 首页增加更明确的阅读顺序提示

文件：`apps/site/src/pages/index.astro`

- 为首页多个 section 补充 `section-note`，把“先读什么、再看什么”直接写出来
- 给“分会董事会 / 近期活动 / 文章”三段增加 `collection-showcase` 容器，减少连续普通卡片区块造成的疲劳感
- 让不同内容集合拥有更稳定的背景层次：分会偏浅、活动偏深、文章偏暖，帮助长页面里的节奏切换更自然
- 修正深色活动展示容器的着色范围，只让头部说明文字变浅，避免内部卡片标题和标签误继承浅色文字

### 2. 三类详情页补一轮 typography 与侧栏收尾

文件：

- `apps/site/src/pages/members/[slug].astro`
- `apps/site/src/pages/events/[slug].astro`
- `apps/site/src/pages/articles/[slug].astro`

主要调整：

- Hero 摘要统一改成更明确的 lead 文本，避免标题之后直接落入普通段落
- 桌面端的成员资料、活动信息、报名侧栏、文章信息侧栏统一改为 `sticky`
- 正文区增加更稳定的最大行长、首段强调和段间距，减少大段文字的“平铺感”
- 说明卡片、附注卡片、相关内容卡片补上更明确的浅色层次，让正文区和辅助信息区更容易区分
- 在 `max-width: 980px` 以下恢复 `position: static`，避免移动端出现黏性侧栏顺序突兀或遮挡问题

### 3. E2E 与截图覆盖扩展到详情页

文件：

- `e2e/site.spec.ts`
- `e2e/screenshots.shots.ts`

本轮补充：

- 新增成员详情、活动详情、文章详情的桌面 / 移动端视口溢出验证
- 把三类详情页加入公开站截图清单，方便后续继续做视觉回看
- 详情页 drill-down 断言改用明确 slug 常量，减少示例数据变化时的维护成本

## 当前结果

完成本轮后，前台公开站点的视觉收尾已经从“页面之间风格统一”进一步推进到“首页长内容节奏更顺，详情页阅读路径更稳”。

当前最明显的收敛点包括：

- 首页不再只是连续堆叠多个内容区块，而是具备更清晰的阅读顺序提示和区块切换节奏
- 成员 / 活动 / 文章详情页都建立了“Hero -> 正文 -> 侧栏补充信息”的稳定阅读模式
- 桌面端的侧栏信息在滚动时更容易持续被看见，移动端则保持自然顺序，不引入额外操作负担
- 详情页已经纳入截图与溢出检查，后续继续做前台视觉微调时更容易回归验证

## 已完成验证

本轮改动后已重新通过以下验证：

- `npm run typecheck`
- `npm run build`
- `npm run test:site`
- `./node_modules/.bin/playwright test e2e/site.spec.ts`
- `./node_modules/.bin/playwright test e2e/screenshots.shots.ts --config=playwright.screenshots.config.ts --grep "capture public site screenshots"`

## 下一轮建议

如果继续推进前台的最后一轮打磨，建议优先考虑：

1. 基于现有截图回看首页和详情页的细节文案密度，进一步删减冗余说明
2. 对卡片图片、头像占位和封面素材再做一轮真实内容替换，减少示例数据感
3. 视需要开始转向后台管理界面的视觉一致性与运营操作体验收敛
