# 前台内容与版式收敛（M2）

## 目标

当前阶段不追求最终视觉包装，而是先把前台七个页面的：

- 信息层级
- 页面节奏
- 关键 CTA
- 不同页面之间的导航关系

整理到可以进入统一视觉精修的状态。

## 页面收敛清单

### 1. 首页

当前重点：

- 首屏已经具备品牌氛围，但页面对“七个核心页面”的承接还不够明确
- 已有分会、活动、文章、加入模块，但“成员网络”仍然偏抽象
- 适合作为组织总览页继续加强导览属性

本轮动作：

- 增加站点导览区，明确七个核心页面分别承担什么作用
- 增加精选成员区，让首页更早出现“真实的人”
- 强化首页从组织介绍 → 分会 → 成员 → 活动 → 文章 → 加入申请的阅读顺序

### 2. 分会董事会

当前重点：

- 页面内容已经完整
- 默认只展开一个分会的交互已经有效降低滚动成本

下一轮动作：

- 进一步优化分会封面、介绍与董事会成员的视觉密度
- 增加“分会差异化”表达，避免所有卡片过于同构

### 3. 成员列表 / 成员详情

当前重点：

- 基本信息完整，但成员页还可以更强地表达“成员网络”而不是“名片墙”
- 成员详情页信息可读，但组织关系与参与路径还可更明确

下一轮动作：

- 成员列表增加更明显的浏览入口与分布感
- 成员详情补强组织位置、参与方式、相关推荐跳转

### 4. 活动列表 / 活动详情

当前重点：

- 活动列表已有分页与城市筛选
- 活动详情报名流程已经可用

下一轮动作：

- 继续优化活动卡片的时间、城市、场地层级
- 优化活动详情首屏与议程区的阅读节奏
- 强化报名说明与审核流程预期

### 5. 文章列表 / 文章详情

当前重点：

- 已经形成“内容服务于社区”的叙事方向
- 但列表和详情仍可进一步拉开节奏，减少模块过于平均

下一轮动作：

- 优化文章列表主次关系
- 优化详情页正文区与侧边说明区比例

### 6. 加入申请

当前重点：

- 已同时承接加入说明与申请表
- 信息已经完整，但还需要更强的“申请前理解 → 提交申请”的转化路径

本轮动作：

- 强化加入说明、申请收益、提交后流程之间的衔接
- 优化表单区与辅助说明区的版式节奏

### 7. 关于我们

当前重点：

- 内容已经覆盖组织方式与活动形式
- 但首屏目前留白偏大，叙事可以更紧凑、更有结构

本轮动作：

- 重构关于页首屏与统计区比例
- 增加“组织原则 / 浏览路径”模块
- 把长文本拆成更清晰的摘要与展开阅读关系

## 本轮实施范围

优先落地以下三页：

1. 首页
2. 关于我们
3. 加入申请

原因：

- 这三页决定访客对平台的第一印象
- 它们承接组织定位、转化路径与品牌叙事
- 变更风险低，不会影响后台或数据流程

## 完成标准

本轮完成后应达到：

- 首页能清楚回答“这是什么组织、有哪些内容入口、下一步看什么”
- 关于页能清楚回答“组织如何运作、为什么这样设计、访客应该继续看哪里”
- 加入页能清楚回答“谁适合加入、加入后得到什么、申请后会发生什么”
