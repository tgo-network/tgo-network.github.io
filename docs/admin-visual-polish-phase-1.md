# 后台统一视觉精修（A1-1）

## 目标

在后台功能链路已经基本可用之后，本轮开始处理第一批后台界面精修，重点不是增加运营能力，而是让工作人员在日常使用时更容易判断：

- 当前自己处在哪个后台模块
- 这一页为什么存在、优先应该处理什么
- 登录、导航、筛选、列表这些高频界面是否具备统一的视觉语言

## 本轮完成

### 1. 后台外壳重新建立工作台感

文件：

- `apps/admin/src/App.vue`
- `apps/admin/src/style.css`

主要调整：

- 为后台增加更完整的 shell 顶栏，显示当前模块、工作说明与工作人员会话状态
- 侧边栏改成更稳定的卡片式结构，补上品牌区、导航分组和会话摘要
- 主区域、按钮、面板、表格、筛选区统一收敛到同一套暖色运营界面语言
- 补强 hover / focus / sticky header 等细节，让表格和导航在日常使用时更稳

### 2. 登录页从“可用表单”升级到“清晰入口”

文件：`apps/admin/src/views/LoginPage.vue`

- 去掉登录页里的内联样式，改成正式布局
- 登录页现在明确说明后台负责的内容范围：内容运营、审核队列、权限边界
- 默认账号提示被收进独立说明卡片，输入区与说明区分层更清楚
- 保留现有 Better Auth 登录流程，不改变认证逻辑

### 3. 管理端视觉回归覆盖补强

文件：

- `e2e/admin.spec.ts`
- `e2e/screenshots.shots.ts`

本轮补充：

- 为登录页增加横向溢出检查
- 管理端截图现在额外包含登录页，便于继续回看后台入口体验
- 继续保留 dashboard / articles / events / applications / members / audit-logs 的截图基线

## 当前结果

完成本轮后，后台已经从“功能先到位”的状态，进入“工作人员可以更顺手使用”的状态。

当前最明显的变化包括：

- 后台外壳不再只是左侧导航加内容区，而是具备更明确的工作台层次
- 登录页不再像临时调试页，而更像正式运营入口
- 列表页、筛选区、按钮和表格在视觉上更统一，后续继续精修单个模块的成本更低
- 管理端现在也有更完整的截图基线，可以继续做后台视觉和运营流程回看

## 已完成验证

- `npm run typecheck`

## 下一轮建议

如果继续推进后台体验收敛，建议优先考虑：

1. 对文章 / 活动 / 申请 / 成员这几类高频列表页继续做密度优化与操作分组
2. 对编辑页补一轮保存状态、预览区和侧栏信息的视觉层级整理
3. 视需要把后台截图覆盖扩展到编辑页和审核详情页
